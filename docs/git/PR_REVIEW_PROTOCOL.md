# PR Review Protocol

Use this if the project is pushed to GitHub or another remote.

## PR Must Include

- Purpose
- Workstream or Orchestrator scope
- Files changed
- Relevant HANDOFF or merge preview
- Contracts touched
- Tests run
- User decisions still needed

## Review Order

1. Path policy
2. Router/schema/memory quality gate
3. Contract consistency
4. Workstream ownership
5. Engine-specific Game Test Gate, if code changed
6. User-facing design intent

## Merge Rule

Merge only after the Orchestrator review is complete and user-required decisions
are resolved.
