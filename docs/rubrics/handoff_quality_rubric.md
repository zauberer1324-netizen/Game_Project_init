# Handoff Quality Rubric

Use this rubric before a workstream says its output is ready for Orchestrator
review.

## Required Qualities

| Criterion | Good Handoff |
| --- | --- |
| Scope | States exactly what the workstream attempted. |
| Inputs | Lists contracts, briefs, and context files read. |
| Proposals | Separates proposed decisions from accepted project facts. |
| Assumptions | Labels unresolved assumptions clearly. |
| Conflicts | Lists conflicts with contracts or other workstreams. |
| Verification | Includes tests, checklists, or review steps. |
| Compactness | Lets the Orchestrator understand the work without reading raw notes first. |

## Minimum Passing Standard

- The Orchestrator can tell what changed.
- The Orchestrator can tell what needs approval.
- The Orchestrator can tell what is risky or unresolved.
- The Orchestrator can decide whether to read detailed artifacts.
