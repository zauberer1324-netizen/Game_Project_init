# Implementation Output Rubric

Use this for code-oriented workstreams and `game_project/` integration.

## Checks

- Is there a PRD, issue, or approved merge preview for the implementation?
- Are module boundaries and interfaces consistent with `code_interface_contract.md`?
- Is the change testable?
- Does the implementation avoid unrelated refactors?
- Does it include a reproduction or verification checklist for bugs?
- Does it run the selected engine profile's Game Test Gate when applicable?

## Output Should Include

- Files changed
- Behavior changed
- Tests or checks run
- Known limitations
- Follow-up issues
