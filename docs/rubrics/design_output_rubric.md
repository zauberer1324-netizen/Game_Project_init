# Design Output Rubric

Use this for character, illustration, storyline, NPC, UI, daily activity, level
design, and progression proposals.

## Checks

- Does it support the current game vision?
- Does it name the contract files it depends on?
- Does it preserve creative alternatives instead of forcing one hidden answer?
- Does it separate flavor, rule, requirement, and open question?
- Does it include enough detail for another workstream to use it?
- Does it avoid editing central memory directly?

## Output Should Include

- Intent
- Player-facing result
- Dependencies
- Proposed decisions
- Open questions
- Risks
- Verification or review checklist
