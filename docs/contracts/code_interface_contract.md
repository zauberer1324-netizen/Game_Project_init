# Code Interface Contract

## Purpose

Define stable implementation contracts between gameplay modules so parallel
workstreams do not invent incompatible interfaces.

## Required Fields To Resolve

- Runtime stack or engine
- Module boundaries
- Player state interface
- Activity system interface
- Progression system interface
- Inventory/resource interface
- NPC interaction interface
- Event naming rules
- Save data shape
- Test harness expectations

## Workstream Rule

Code-oriented workstreams may propose interfaces, but the Orchestrator accepts
or rejects them before they become shared contracts.
