# Game Vision Contract

## Purpose

Define the stable high-level target that all workstreams must align with.

## Current Status

The final genre, engine, camera, art style, and core loop are not yet locked.
Workstreams must avoid assuming details that have not been accepted into
`CONTEXT.md` or this contract.

## Required Fields To Resolve

- Game genre
- Target platform
- Camera perspective
- Core player fantasy
- Primary loop
- Secondary loops
- Tone and mood
- First playable prototype scope
- Explicit out-of-scope features

## Workstream Rule

If a workstream needs a vision detail that is not resolved, it must list the
assumption in `HANDOFF.md` and propose a context update instead of silently
inventing the answer.
