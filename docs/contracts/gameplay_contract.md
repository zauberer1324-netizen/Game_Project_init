# Gameplay Contract

## Purpose

Define shared gameplay rules used by mechanics, physics, NPC, UI, level design,
and daily activity workstreams.

## Required Fields To Resolve

- Core loop
- Player state model
- Input model
- Time model
- Stamina, fatigue, or stress model
- Risk and failure model
- Reward categories
- Save/load expectations
- Debug and verification expectations

## Workstream Rule

Gameplay-facing proposals must specify the player-visible behavior, failure
conditions, and verification checklist.
