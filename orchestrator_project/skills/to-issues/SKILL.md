# To Issues

Break work into thin vertical slices. Each issue should be independently
verifiable and should cut through all needed layers for one narrow behavior.

## Issue Fields

- Title
- Type: `AFK` or `HITL`
- Blocked by
- What to build
- Acceptance criteria
- Out of scope

Prefer many small demoable slices over one broad horizontal task.
