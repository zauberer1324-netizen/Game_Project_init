# To PRD

Create a PRD from the current conversation and codebase understanding. Do not
interview unless required information is missing.

## Template

1. Problem Statement
2. Solution
3. User Stories
4. Implementation Decisions
5. Testing Decisions
6. Out of Scope
7. Further Notes

Use domain language from `CONTEXT.md`. Avoid brittle file paths in the PRD.
