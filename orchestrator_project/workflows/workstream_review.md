# Workstream Review Workflow

Use this workflow when one or more workstreams finish a proposal and need
Orchestrator review before central integration.

## Steps

1. Identify reviewed workstreams and review scope.
2. Read contracts and handoffs before detailed outputs.
3. Run the vision-fit checkpoint.
4. Directly classify mechanical, contract, file, and ownership conflicts.
5. Produce proposal and conflict tables.
6. Propose integration updates.
7. Confirm before editing central memory or implementation.
8. Update Orchestrator memory and write a review report when approved.

## Output

- Vision-fit status
- Accepted proposals
- Revision requests
- Rejected proposals
- Conflict table
- Integration plan
- Orchestrator memory update
